How is the Myanmar language encoded?

A: The Myanmar script has been included in Unicode since Version 4.0 (VERIFY) and was significantly enhanced in Version 5.1. Three blocks are defined for characters of this script, including the base characters (Range: 1000-109F), Myanmar Extended-A (Range: AA60–AA7F), and Myanmar Extended-B (Range: A9E0–A9FF). These support several languages including Myanmar (Burmese), Pali, Sanskrit, Shan, Mon, Karen, Kayah, and others.

What other encodings are commonly used ?

Several other encodings are in common use, all based on different font mappings. These include ZawgyiOne, Zawgyi 2000, Myazedi, and others.

What are the differences between the encodings?

Most encodingsare not compatible with Unicode, although many use the same code point range as the base Unicode block (U_1000 - U+109F). Some characters are common in these fonts, especially the base consonants, but other characters and combining marks differ widely. This has several implications:

  • UTF-8 can be used to represent the characters of any of these fonts, including Unicode and Zawgyi, which is the the most common non-Unicode scheme.
  • The characters displayed depend on the font that is used.
  • Displaying Characters with a different font than the encoding will appear to be corrupted, typically showing "dotted" characters disconnected from base consonants, and also and incorrect characters.   • Since the code points overlap among these encodings, it is impossible to create a "universal font" that will display different encodings at the same time.
  • It is important to detect the encoding used in Burmese text in order to to apply the appropriate font.
  • Machines of the same type or with the same operating system may have different Myanmar fonts and keyboards installed.
  • Text is not automatically converted by most systems. To be understood, text must be shown in the same font as sent, or the text must be converted to the reader's font.
  • To display text consistently, characters should be converted to a single encoding such as Unicode, then use a Unicode-compliant font.

How can I tell what encoding is used for a particular website or piece of text?

Almost all text in a given encoding will render correctly, i.e., without corrupted rendering, only when displayed with a compatible font. For example, Zawgyi text will appear incorrect with a Unicode font, and text encoding as Unicode will look wrong with ZawgyiOne font.

Some online tools are available that will help determine the encoding of text.

  • zawgyi-unicode-test.appspot.com takes text and displays it using several common fonts, including two diiferet Unicode fonts.
  • Several web sites offer detectors that use Javascript to look for non-Unicode patterns. Give some examples and links.
