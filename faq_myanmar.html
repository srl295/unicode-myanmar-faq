How is the Myanmar language encoded?

A: The Myanmar script was added to the Unicode Standard in Version 3.0 (September, 199) and was significantly extended in Version 5.2 (2009). Unicode now has three blocks for characters of this script:

  • The base Myanmar characters (Range: 1000-109F),
  • Myanmar Extended-A (Range: AA60–AA7F),
  • Myanmar Extended-B (Range: A9E0–A9FF).

Code points include letters (consonants and independent vowels), vowel signs, medial signs, digits, various signs, and punctuation. Medial and vowel signs, anusvara, visarga, virama, asat and others combine with letters. Unicode defines the order of combining characters for correct display.

These code points support several languages written in the script, including Myanmar (Burmese), Pali, Sanskrit, Shan, Mon, Karen, Kayah, and others.

What other encodings are commonly used ?

A: Several other encodings are in common use, all based on different arrangements of characters and combining marks. These include ZawgyiOne, Zawgyi 2000, Myazedi, and others.

What are the differences between these encodings?

A: Although other encodings may use the same code point range as the base Unicode block (U_1000 - U+109F), they are not compatible with Unicode. Some characters are common in these fonts, especially the base consonants, but other characters and combining marks differ widely. Of course, the shapes shown for given code points depend on the specific font that is used.

Wikipedia:Font shows the code point differences for both Unicode and Zawgyi, which is the most commonly used non-Unicode scheme.

For each combining character, Unicode defines a single code point such as U+103c, which is the ra medial that surrounds a consonant with a line. A Unicode font combines this with other code points to create the right shape.

Unicode also defines a unique order of code points for base letters adn combining characters.

Non-unicode fonts use as many as 8 code points for different parts of the ra glyph for narrow and wide consonants and other conditions. The user needs to type the right one for each context. This also means more code points are needed for the Burmese language than in the Unicode. These fonts also allow multiple orderings of combining marks and base letters, without a unique representation of each combination.

An incorrect match between font and text shows "dotted" characters or overlapping lines, and also and incorrect characters, as shown in the following table.

To be understood correctly, text must be shown in the same font as written, or the text must be converted to the font used for display.

Can we use a universal font that will display Unicode and Zawgyi text together?

A: No. Since the code points of different characters are in the same range (0x1000-0x109f), no font can automatically apply the right character shapes. No universal font is possible.

However, HTML may use different fonts explicitly in tagged regions if the encoding of the text inside has been detected.

Doesn't "UTF-8" indicate Unicode?

A: No. UTF-8 is a scheme for representing a code point in 1, 2, 3, or 4 bytes. Since the code points all use the same range, UTF-8 can be used to represent any of these encodings.

What are some advantages and disadvantages of different encodings?

A: Here are the main Unicode points:

  • + It is the standard accepted by the World Wide Web Consortium, and works on almost every device and operating now available.
  • + Search is seamless with Unicode, since there is only one way to define a character.
  • + Unicode fonts support more than 12 languages written in Myanmar script, not only Burmese.
  • + Each combining mark is a single character, making input easier.
  • + Multiple Unicode-compatible fonts are available, allowing users to choose different "look and feel", similar to other scripts such as Latin.
  • - The font rendering is more complex, requiring a modern rendering engine.
  • - Code points must be in a defined order for combining marks to be shown.
  • - Many websites, documents, and systems have been implemented with other encoding schemes.

The following notes apply to ZawgyiOne, Zawgwyi 2008, Myazedi and others:

  • + Widely used for existing web sites, documents, and user communictions.
  • + A simpler rendering engine can implement the graphical output.
  • - Many more code points are used, so only the Burmese language can be written with these fonts. Characters for other languages using this script are missing.
  • - In most cases, only a single font has been defined for each encoding, limiting styles of text that can be displayed.
  • - The order of code points is flexible, making searching and matching text difficult. For example, search engines results are highly dependent the specific order in which text has been typed.

How can I tell what encoding is used for a particular website or piece of text?

A: Almost all text in a given encoding will render correctly only when displayed with a compatible font. For example, Zawgyi text will appear incorrect with a Unicode font, and text encoding as Unicode will look wrong with ZawgyiOne font. However, since a subset of characters is common to all these fonts, some strings will be identical in all encodings.

Some online tools are available that will help determine the encoding of text.

  • zawgyi-unicode-test.appspot.com takes text and displays it using several common fonts, including two diiferet Unicode fonts.

So isn't Unicode just another font for Myanmar?

A: Yes and no. Unicode for Myanmar is more than a font. It defines character code points and also requires a specific ordering of code points for consistent text rendering. Unicode-compliant fonts already expect this ordering to display characters correctly.

As a published standard, Unicode provides a public description of each code point, plus characterstics that can be used for other text processing functions such as its collation, combining status, and combining order. These are available via the ICU C++ and Java libraries.

Should I support both Unicode and Zawgyi on my site? If so, how do I do that?

A: Since may platforms do not yet have Unicode fonts, it is very helpful to provide a way for any user to view content. Several techniques are described here:

  • Use a webfont on your site and apply it with CSS in any HTML block that displays text, e.g., a <div class="myfontclass"> tag. The font is loaded along with the text, allowing modern browsers to display text in the loaded font. This works well in most cases for either Zawgyi or Unicode text. However, transmitting the font increases load time for such content.
  • Let users switch via a prominent control on the page to select either Unicode or other encoding. Then use this setting to load load the correct pages. An automatic converter may be used to prepare text as needed. This has the advantage of no font download, but adds complexity on both the client and server.
  • Detect the encoding of user-entered text. Use this to provide text in the detected encoding.
  • Don't worry about it. Provide content in only Zawgyi or Unicode and let users determine to use your site based on the encoding. This limits the visibility of your content, of course.
Important: Remember that search engines may not understand all text encodings. Unicode text on your site can be consistently interpreted.

My site has content entered by users in both Zawgyi and Unicode text. How can my users read both?

A: Good for you to care that your users can read all messages! There are at least two ways to handle this, similar to the methods described for web sites. Each requires detecting the encoding of each message posted.

  • Use webfonts for the site. Make sure each posting is in its own tagged block such as div. Set the CSS for each posting to a font based on the encoding.
  • Convert all postings to Unicode form. Set CSS to use Unicode-compatible fonts.
You may consider educating your users on using Unicode fonts.

How is Myanmar handled on mobile devices?

A: Most mobile devices do not allow the user to change or replace the installed fonts. An application may "bundle" a font, but that will only be used within the application, not for other tools or apps.

Many devices already include a Unicode compliant font that is used by default for any Myanmar text. Any Unicode text will be appear correctly in the system-installed applications. Zawgyi text will be shown incorrectly unless the particular application has included the Zawgyi font within the application.

Some phone vendors have installed ZawgyiOne in place of a Unicode font. In this case, Zawgyi will look right, but Unicode text in messages and web sites will look wrong.

It is possible for an application to detect and convert text between encodings to match the installed fonts. However, this is not common.

My friends all use Zawgyi in email and texting, but my device only supports Unicode. How can I communicate with them?

A: This is more complicated because fonts cannot be added or changed on most mobile devices. Free Myanmar Unicode keyboards are available for most mobile devices from online sources, so work with your friends to agree on a common way to communicate.

Apps that convert between Zawgyi and Unicode are also available. Copying and pasting text messages into such an interactive converter will let you read any message.

I cannot find the code points for the kinzi in Unicode. What do I do?

A kinzi is where the first consonant in a cluster is a non-word-final letter NGA. In this case, NGA rises over the following letter, as described here. This combination appears as a small character similar to a Greek epsilon over the following letter.

ZawgyiOne represents kinzi with a single code point 0x1064 for this combination kinzi

.

In Unicode, NGA + ASAT is entered, followed by a VIRAMA to position the character over the next letter. This allows searching for the NGA letter as a character. The kinzi sequence is code points U+1004 U+103A U+1039, followed by the next letter.
